## Description

**scMultiome-GRN** is a Nextflow DSL2 pipeline that integrates Single-Cell Multiome data
(scRNA-seq + scATAC-seq), infers Gene Regulatory Networks (GRNs) and simulates *in silico*
transcription-factor perturbations with vector-field visualization.

The pipeline is built around a native **polyglot** design:

- **R** (`Seurat`, `Signac`, `cicero`) — chromatin accessibility preprocessing and
  co-accessibility analysis.
- **Python** (`Scanpy`, `CellOracle`, `GimmeMotifs`) — base-GRN construction, network
  scoring and in silico perturbation.

It is surgically optimized for a 1-hour live-demo masterclass: it runs an ultra-downsampled
public dataset on a standard laptop in under 2 minutes, spectacularly demonstrating the
power of Nextflow's caching system (`-resume`).

## Quickstart

Run the pipeline with the bundled biological test dataset (microglia from
the 10x Multiome Mouse Brain Alzheimer's AppNote):

```bash
# 1) (one-time) obtain and subsample the test dataset
bash  data_demo/download_data.sh        # ~30 min, ~6.5 GB into data_demo/raw/
python data_demo/subsample_dataset.py   # ~10 min, produces data_demo/processed/

# 2) Pull or build the container image
docker pull ghcr.io/dmouzo/scmultiome-grn:latest

# 3) Run the full pipeline
nextflow run main.nf -profile test,docker

# 4) Re-run with a different network-pruning percentile
#    (the heavy R preprocessing step is CACHED; only Python re-runs)
nextflow run main.nf -profile test,docker --percentile 95 -resume
```

For CI smoke tests, point `--input` to `data_demo/synthetic/mini_seurat.rds`
(an ultra-tiny synthetic object regenerated by
`data_demo/synthetic/build_synthetic_mini_seurat.R`).
